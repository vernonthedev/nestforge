# Core Concepts

## Module

A module wires the app together.

It declares:

- controllers (HTTP entry points)
- providers (services/config in DI)

In code, this is the `ModuleDefinition` contract generated by `#[module(...)]`.

## Controller

A controller owns route handlers.

- `#[controller("/users")]` sets the base path.
- `#[routes]` builds a router from method attributes.

## Service / Provider

A provider is a value registered in DI.

Examples:

- `UsersService::new()`
- `AppConfig { app_name: ... }`

## Dependency Injection (Container)

`Container` stores providers by type.

- `register<T>(value)` adds one instance
- `resolve<T>()` gets it back as `Arc<T>`

## Request helpers

- `Inject<T>`: get a provider from DI in a handler
- `Param<T>`: read route params
- `Body<T>`: read JSON body

## HTTP errors

Use `HttpException` to return JSON errors with status code + message.

## In-memory store

`InMemoryStore<T>` is a simple shared store for demo apps.

`T` must implement `Identifiable`.
